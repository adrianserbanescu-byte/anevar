# Audit de securitate — Evaluare ANEVAR

Aplicație desktop offline (Windows): FastAPI + Jinja2 + SQLite + foldere, livrată ca `.exe` (PyInstaller). Gestionează PII (CNP, nume client, adrese), generează documente AML/juridice, importă `.docx`/PDF, descoperă comparabile pe portaluri, are o extensie de browser, backup `.zip` și un AI gateway (parțial implementat: Anthropic/Perplexity).

**Model de amenințare.** Serverul ascultă pe `127.0.0.1:8000` (loopback), fără autentificare. Atacatorii relevanți NU sunt din rețea, ci: (a) **pagini web** vizitate de evaluator (CORS + CSRF + DNS-rebinding pe localhost), (b) **conținut extern controlat de atacator** procesat de aplicație (URL-uri de anunț → SSRF; HTML de portal; fișiere `.docx`/PDF importate), (c) **fișiere de dosar** importate din surse externe. Pe acest model, mai jos sunt vulnerabilitățile reale.

Calea fișierului: `C:\Users\adyse\anevar\docs\audit-final\security.md`

## Verificări cerute — rezultat

- **(1) Path traversal** (import-docx/anexe/backup): **Mitigat.** `import-docx` folosește `Path(req.nume_fisier).name` într-un tmpdir unic (`curent.py:75-78`); numele de folder dosar e mereu `uuid4()` (`dosare_fs.py:43`), iar `nume_dosar` produce doar un string de afișare, nu o cale. AML store folosește index numeric controlat (`store.py:45`). Niciun traversal exploatabil.
- **(2) SSRF** (import-url/descopera): **CONFIRMAT** — vezi #1. `descopera`/`descopera_teren` construiesc URL-uri din `BAZE` fixe, dar fetch-ul URL-urilor de anunț extrase din HTML (`orchestrator.py:118,178`) și `import-url` direct nu validează hostul.
- **(3) ZIP backup** (symlink/.env): `.env` **NU** e inclus (în afara arborelui zipat). Symlink/junction-following = risc minor #7.
- **(4) Injecție** `md_to_html`/docx/nume folder: `md_to_html` escapează corect (#5, latent). Nume folder = uuid (fără injecție). Niciun `eval`/`exec`/`os.system`/`subprocess`/`pickle`/`yaml.load` în cod (grep curat).
- **(5) Bind server:** `127.0.0.1` — **corect** (#8).
- **(6) PII în loguri/payload AI:** Loguri OK (se loghează doar hash-uri/erori, nu PII brut). Payload AI — scurgere parțială (#6).
- **(7) base64 fără limită (DoS):** **CONFIRMAT** — #3.

**Pozitive:** SQL 100% parametrizat (`db/storage.py`) — fără SQL injection. Bind loopback. Anonimizare prezentă (deși incompletă). Scriere atomică a fișierelor. CORS fără `allow_credentials`.

## Top 5 (prioritizat)

1. **#1 SSRF la `/api/import-url`** — *EXPLOATABIL.* Validare URL + blocare IP interne/metadata + control redirect. (Bucket A)
2. **#2 CSRF / DNS-rebinding pe API-ul local fără auth** — *EXPLOATABIL.* Verificare `Host` header (anti-rebinding) + token/header custom pe POST-uri de scriere. Amplifică #1, #3, #7. (Bucket A)
3. **#3 DoS prin base64 nelimitat** (import-docx / ingestie PDF / foto) — *EXPLOATABIL.* Limită de dimensiune pe request + pe base64. (Bucket A)
4. **#6 Scurgere PII la AI** (`beneficiar` în clar, regex CNP incomplet) — *EXPLOATABIL ca GDPR*, doar cu gateway activ. Extinde anonimizarea. (Bucket C)
5. **#4 Import folder dosar fără validare** — *Teoretic→Mediu.* Validare schemă + filtrare tipuri/mărime fișiere. (Bucket B)
